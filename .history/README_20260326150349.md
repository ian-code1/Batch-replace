# Batch Search and Replace Tool

A safe and flexible tool that helps you find and replace text across multiple files at once. Perfect for updating codebases, changing configurations, or cleaning up text files.

## Why Use This Tool?

- **Safe by design** - Creates copies of your files, never touches the originals
- **Smart file detection** - Automatically works with text files, skips binary files
- **Flexible rules** - Single replacements or complex batch operations
- **Environment ready** - Different rules for development, staging, and production
- **Preview mode** - See what changes before applying them
- **Works everywhere** - Windows, Mac, and Linux

## Quick Start

### Installation

```bash
# Download and install
git clone https://github.com/yourdepartment/batch-replace-tool.git
cd batch-replace-tool
pip install -e .
Simple Example
Replace "Hello" with "Hi" in all text files:

bash
batch-replace --source ./myproject --destination ./output --pattern "Hello" --replace "Hi"
The tool will:

Copy all files from myproject to output

Replace every "Hello" with "Hi" in text files

Leave your original files unchanged

Common Tasks
1. Work with Specific File Types
Only change JavaScript and HTML files:

bash
batch-replace --source ./src --destination ./out \
  --pattern "localhost" --replace "127.0.0.1" \
  --file-types .js,.html
2. Use Regular Expressions
Find and mask phone numbers:

bash
batch-replace --source ./logs --destination ./clean-logs \
  --pattern "\b\d{3}-\d{4}\b" --replace "[PHONE]" --regex
3. Preview Changes First
Always test before making changes:

bash
batch-replace --source ./src --destination ./out \
  --pattern "old" --replace "new" --dry-run
This shows what would change without actually modifying anything.

Advanced Features
Multiple Replacements with a Rules File
Instead of running the tool multiple times, create a rules file:

rules.json

json
{
  "rules": [
    {"pattern": "old-domain.com", "replace": "new-domain.com"},
    {"pattern": "localhost:3000", "replace": "api.example.com"},
    {"pattern": "password\\s*=\\s*['\"](.*?)['\"]", "replace": "password=\"[REDACTED]\"", "regex": true}
  ]
}
Then run:

bash
batch-replace --source ./config --destination ./new-config --rules-file rules.json
Environment-Specific Settings
Use different settings for development and production:

env_config.json

json
{
  "environments": {
    "dev": {
      "API_URL": "http://localhost:8000",
      "DB_HOST": "localhost"
    },
    "prod": {
      "API_URL": "https://api.example.com",
      "DB_HOST": "prod-db.example.com"
    }
  }
}
rules.json (using placeholders)

json
{
  "rules": [
    {"pattern": "api.example.com", "replace": "${API_URL}"},
    {"pattern": "localhost:5432", "replace": "${DB_HOST}:5432"}
  ]
}
Run for different environments:

bash
# Development
batch-replace --source ./config --destination ./dev-config --rules-file rules.json --env dev --env-config env_config.json

# Production
batch-replace --source ./config --destination ./prod-config --rules-file rules.json --env prod --env-config env_config.json
Skip Unwanted Folders
Ignore common directories:

bash
batch-replace --source ./project --destination ./output \
  --pattern "old" --replace "new" \
  --ignore-dirs .git,node_modules,dist,__pycache__
Keep Original Timestamps
Preserve file creation and modification dates:

bash
batch-replace --source ./src --destination ./out \
  --pattern "old" --replace "new" --preserve-timestamps
All Available Options
Option	What it does	Example
--source	Folder to read files from	--source ./myproject
--destination	Folder to write modified files	--destination ./output
--pattern	Text to search for	--pattern "localhost"
--replace	Text to replace with	--replace "127.0.0.1"
--rules-file	File with multiple rules	--rules-file rules.json
--regex	Use regular expressions	--regex
--file-types	Only process these file types	--file-types .js,.css
--ignore-dirs	Skip these folders	--ignore-dirs .git,node_modules
--dry-run	Preview without changing files	--dry-run
--env	Environment name	--env production
--env-config	Environment settings file	--env-config env.json
--preserve-timestamps	Keep original file dates	--preserve-timestamps
--verbose	Show detailed information	--verbose
--log-file	Save log to file	--log-file mylog.txt
Real-World Examples
Example 1: Update API Endpoints
Change all API URLs in your JavaScript files:

bash
batch-replace \
  --source ./src \
  --destination ./src-updated \
  --pattern "https://old-api.example.com" \
  --replace "https://new-api.example.com" \
  --file-types .js,.ts,.html
Example 2: Remove Debug Code for Production
Clean up console.log statements and debuggers:

cleanup-rules.json

json
{
  "rules": [
    {"pattern": "console\\.log\\(.*?\\);?", "replace": "", "regex": true},
    {"pattern": "debugger;", "replace": "", "regex": false}
  ]
}
bash
batch-replace \
  --source ./src \
  --destination ./dist \
  --rules-file cleanup-rules.json \
  --env prod \
  --env-config env_config.json
Example 3: Mask Credit Card Numbers in Logs
Find and hide credit card numbers before sharing logs:

bash
batch-replace \
  --source ./logs \
  --destination ./logs-sanitized \
  --pattern "\\b\\d{16}\\b" \
  --replace "[CREDIT_CARD]" \
  --regex \
  --dry-run
Run without --dry-run when you're ready to apply changes.
